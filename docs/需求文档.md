我要创建一个小软件，具体用什么架构还不知道，目前待定。这个软件有点类似于 CC switch，但是功能不太一样。

主要功能如下：
1. API 管理与配置：
软件会管理我的很多大模型 API。通过在项目根目录下的 JSON 文件进行配置（`cfgs/app.json` 配置服务和鉴权，`cfgs/model_api.json` 配置上游 profile），服务通过 `app.json` 中统一的 `local_key` 做本地鉴权，再按客户端请求路径的协议家族选择默认 profile：`/v1/chat/completions` 走 `default_openai_model`，`/v1/messages` 走 `default_anthropic_model`，其他生成类端点返回 404 `UNSUPPORTED_ENDPOINT`；每个 profile 用 `model_name` 作为本地标识（用于与两个默认字段匹配），并用 `model_id` 作为真正下发到上游 API 的模型名，自动读取供应商的 API 链接以及最大 token、最大输出 token 等参数。

2. 固定 IP 转发：
软件会将这些 API 通过一个本机固定入口转发出去，默认监听 `127.0.0.1:8787`；`host` 可在 `app.json` 中改为 `localhost` / `0.0.0.0` / 局域网 IP，`port` 同样可配。改成 `0.0.0.0` 或局域网 IP 时，同网段任意设备只要知道 `local_key` 就能调用真实上游 API，需要搭配足够强的 `local_key`。

3. 编程工具集成：
最后，我希望在 TRAE 或其他编程工具里面可以直接使用。我在编程工具里面直接输入它转发的地址（默认 `http://127.0.0.1:8787/v1`）以及本地 token（对应 `app.json` 的 `local_key`），从而实现在编程软件里面直接使用转发出来的大模型 API，我就不用在编程软件里面去填真实 API Key 了。

4. 桌面托盘运行：
下一轮迭代希望软件运行后以 Windows 右下角任务栏托盘小图标的形式常驻。托盘版默认不显示命令行窗口，但需要保留调试模式。托盘右键菜单按 OpenAI / Anthropic 两个协议家族分组，列出所有 `enabled: true` profile，以 radio 单选形式切换默认模型（`default_openai_model` / `default_anthropic_model`），切换后写回 `cfgs/app.json` 并热更新内存运行时，无需重启进程；菜单末尾始终保留“退出”项，退出时同时关闭本机代理服务。托盘版作为可双击运行的 Windows 单文件便携 exe 交付，与 `cfgs/` 目录同级放置即可开箱使用；开发态运行时沿用 CLI 从工作目录读取配置的规则，打包 exe 运行时改为从 exe 同级 `cfgs/`、`logs/` 读写。
